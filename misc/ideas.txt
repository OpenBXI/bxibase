- Interpréteur de bxilog (à base de ncurses ou autres)
- Logger name with short versions (for console)
- posless: --verbose, --Debug -> they use predefined configuration files.
- bxilogparser --from now -15mn: seek in the logging files in order to start with the good date
- bximonitor: un handler de log capable de retourner une configuration clé/valeur et qui permet de faire des get/set (à distance)
  par exemple -> changer la configuration d'un ou des loggers
- bxilog_handler -> en faire des plugins
- bxilog_handler -> pushing to logstash
- bximonitor -> ncurses
- bxilog : bitmap de configuration (fonction, event, configuration, message, ... à la nagios). Pour les fonctions, proposer des macros: IN(level, args) et OUT(level, return)
- last message : pour chaque niveau, retourne le dernier message reçu (bximonitor last)
- bxilogparser : regroupe les logs similaires (même ligne, même fichier)
- monitoring temps réels des threads bloquées : lors de l'envoi d'un EAGAIN, on incrémente un compteur de manière atomique puisqu'on a le temps. La thread de monitoring augmente le HWM ou le diminue en fonction
